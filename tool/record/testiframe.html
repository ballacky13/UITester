SNS-JSSDK

概述

SNS-JSSDK是基本淘宝SNS的前端组件开发工具，目的是为淘宝以及外部网站 提供具有淘宝SNS特性的功能。这些功能包括丰富的SNS组件，如关注，喜欢，分享等， 还可以通过JSSDK方便的调用淘宝SNS http接口。如果现有的组件 无法满足你的要求，你可以使用这些http接口获取数据，编写自己特有的功能。

SNSDK提供了多种组件的调用方式，保障组件调用的简单和功能的安全性。这些方 式包括普通DOM标签方式， iframe标签方式，JSAPI方式。这些方式的实现都是SDK自动完成的。组件使用者只需要做一些简单的配置就可以在不同的方式之间切换 。

如何使用

SNSDK是使用模块化和异步加载的方式实现的，分为两个主要部分。 第一部是核心部分，提供了一些重要的基本方法，必需提前引入。第二部分是异步加载的部分，当调用组件 的时候才会被加载到页面中.另外SNSDK是在KISSY(1.1.6 or 1.2.0)的基础上实现。

第一步：引入KISSY

<script type="text/javascript" src="http://a.tbcdn.cn/s/kissy/1.2.0/kissy.js"></script>

第二步：引入核心JS

<script type="text/javascript" src="http://a.tbcdn.cn/p/snsdk/core.js></script>

第三步：调用组件

<div  class="sns-widget" data-name="configs">淘宝SNS组件</div>

name:代表组件的名称， configs:是一个json格式的字符串，代表组件的配置,有时候可省略

例子

<div  class="sns-widget" data-like>这是一个喜欢组件</div>

这是一个喜欢组件

SDK通用配置

配置appKey(即将实现)，目前系统会自动生成一个client_id, 不需 要申请

SNS.config("appKey",123123)

组件通用配置

样式配置

默认情况下，SNS组件会默认提供一些样式，如果你不想使用默认样 式，可通过配置skin参数的方式来修改。如果设为null，调用组件时将不会加载css文件; 如果设为其它地址，调用组件时将会自动替换为当前地址; 

//调用时配置
<div class="sns-widget" data-like="{ui:{skin:'./newcss.css'}}">这是一个喜欢组件</div>

这种配置也可以再调用组件时进行配置

//调用前配置
SNS.config("widgets",{
    name:"like",
    skin:null
})

更新配置（即将实现）

默认情况下SNS组件都会自动更新，如果你不希望自动更新，或者自 己选择什么时候更新，可以通过配置时间截参数来修改。如果不设置，调用组件时会默认自动判断有无 更新，如果设置了时间截，将使用当前的时间截进行更新，这种配置也可以再调用组件时进行配置

//调用时配置
<div  class="sns-widget" ></div>
//调用前配置
SNS.config("widgets",{
    name:"like",
    timestamp:2012413
})

版本配置

某些SNS组件可能有很多不同版本，不同的版本之前有一些功能的差 异，你可以通过配置version的方式，选择不同的版本。

//调用时配置
<div  class="sns-widget" data-like="{ui:{version:2.0}}">这是一个喜欢组件</div>
  
 //调用前配置
 SNS.config("widgets",{
    name:"like",
    version:2.0
})

性能配置

当你大量使用SNS组件的时候，页面的脚本是会变的非常多，这可能 在一定程度上影响了页面的性能，我们也提供了一些方式可以合并这些请求

 //将一个组件和多个组件的请求合并
 SNS.config("widgets",{
    name:"like",
    combo:["share","comment"]
})

//可能通过通配符*合并请求
SNS.config("widgets",{
    name:"*",
    combo:["like", "share","comment"]
})

调用组件的多种方式

第一种：标签方式

在某个DOM标签上做适当的配置，SNSDK会在页面渲染完成之后自动 发现这些配置，初始化组件。使用这种方式无需编写JS代码，简单方便。适用于对于自定义要求不高的 组件，或者无法注入脚本的情况。如 由于旺铺中无法注入脚本，旺铺中必须使用这种方式

<script type="text/javascript" src="http://a.tbcdn.cn/s/kissy/1.2.0/kissy.js"></script>
<script type="text/javascript" src="http://a.tbcdn.cn/p/snsdk/core.js></script>
<div class="sns-widget" data-like="{}">这是一个喜欢组件</div>

第二种：JS-API方式

某些情况下，可能需要动态的创建组件，这时候你可能需要使用JS- API的方式调用组件

 <script type="text/javascript" src="http://a.tbcdn.cn/s/kissy/1.2.0/kissy.js"></script>
<script type="text/javascript" src="http://a.tbcdn.cn/p/snsdk/core.js></script>
<div " id="sns-widget-like">这是一个喜欢组件</div>
<script>
    SNS.ui("like",{
            element:"#sns-widget-like"
    })
</script>

iframe组件

使用iframe组件，可以保障组件不受第三方页面的影响， 组件也不会影响外部页面。部分组件由于安全性的原因只能在淘宝的页面中使用，第三方站点只能通过 iframe的方式才能使用这些组件。因些 使用iframe的方式可以使组件部署的任何页面中。 建议非淘宝页面使用iframe方式。

方式一：在页面中配置一个iframe标签

 <script type="text/javascript" src="http://a.tbcdn.cn/s/kissy/1.2.0/kissy.js"></script>
<script type="text/javascript" src="http://a.tbcdn.cn/p/snsdk/core.js></script>
<iframe  class="sns-widget" data-like="{url:'http://www.taobao.com'}">这是一个喜欢组件</iframe>

方式二：使用JSAPI， 设置当前的元素为一个iframe标签

<script type="text/javascript" src="http://a.tbcdn.cn/s/kissy/1.2.0/kissy.js"></script>
<script type="text/javascript" src="http://a.tbcdn.cn/p/snsdk/core.js></script>
<iframe id="sns-widget-iframe-like">这是一个喜欢组件</iframe>
SNS.ui("like",{
    ui:{
        width:100,
        height:100,
        element:"#sns-widget-iframe-like"
    }
})

使用SNSDK httpAPI

此API封装了淘宝SNS各种HTTP接口，可能以更直观更简单的http方式 ，存取淘宝SNS的数据，如果现在组件无法满足你的要求，httpAPI将非常有帮助

API

SNS.api(url, data, callback)

例子

SNS.api("/like",{url:"http://wwww.taobao.com"}, function(data){SNS.alert(data.result.content)})
